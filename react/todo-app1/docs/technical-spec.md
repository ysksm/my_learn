# 技術仕様

## フレームワーク

- React 19

## アーキテクチャ

- **レイヤードアーキテクチャ**: アプリケーションを複数の層に分離
- **ドメイン駆動設計（DDD）**: ビジネスロジックをドメイン層で表現

## アーキテクチャ原則

- **依存性逆転の原則（DIP）**: アーキテクチャの構成を維持し、上位層が下位層の実装に依存しないようにする
- **依存性注入（DI）**: レイヤードアーキテクチャとドメイン設計の一貫性を保つ

## 実装方針

### リポジトリパターン

- ドメイン層でリポジトリのインターフェースを宣言
- インフラ層でリポジトリの具体的な実装を提供
- DIを通じてドメイン層やアプリケーション層にリポジトリを注入
